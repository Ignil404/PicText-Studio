## Why

Текущее приложение имеет только клиентскую часть для создания изображений, но отсутствует интерфейс управления для администраторов. Необходима админ-панель для управления шаблонами, просмотра статистики использования и управления пользователями без прямого доступа к базе данных.

## What Changes

- **Новая админ-панель** (`/admin`) с авторизацией по роли admin
- **Управление шаблонами**: CRUD операции (создание, редактирование, удаление, просмотр)
- **Статистика использования**: количество рендеров, популярные шаблоны, активность пользователей
- **Управление пользователями**: просмотр списка, блокировка/разблокировка, сброс пароля
- **API endpoints** для всех операций с проверкой прав admin
- **AdminGuard** на фронтенде — защита роутов по роли

## Capabilities

### New Capabilities
- `admin-dashboard`: Главная страница админ-панели с обзорной статистикой
- `admin-templates`: Управление шаблонами (CRUD, preview, активация/деактивация)
- `admin-users`: Управление пользователями (список, поиск, блокировка)
- `admin-stats`: Статистика и аналитика использования приложения
- `admin-auth`: Ролевая модель (admin role) и защита админ-эндпоинтов

### Modified Capabilities
- `user-auth`: Добавить поле `role` (user/admin) в модель User и JWT токен

## Impact

- **Backend**: Новые роутеры в `backend/routers/admin/`, middleware для проверки роли admin
- **Frontend**: Новые страницы в `frontend/src/pages/admin/`, защищённые роуты
- **Database**: Миграция добавления `role` column в таблицу users
- **Dependencies**: Новые компоненты shadcn/ui для таблиц (data-table), форм, модалок
- **Security**: Все admin endpoints требуют JWT с ролью admin
